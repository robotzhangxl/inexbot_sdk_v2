# 7000端口协议

7000 端口是控制器与上位机之间的主要通讯接口，支持 **TCP** 和 **UDP** 两种通讯协议。

## 通讯配置

在控制器配置文件 `config/controller.json` 中的 `hostComputerService` 节点进行配置：

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| socketType | string | TCP | 通讯类型，可选 `TCP` 或 `UDP` |
| checkCRC | bool | true | 是否进行 CRC 校验 |

> **注意：** 即使关闭 CRC 校验，CRC 占用的 4 个字节仍需发送。

> **性能提示：** 查询状态时使用 UDP 通讯，最小周期可达 **1ms**。

## 协议帧格式

| 字段 | 长度 | 说明 |
|------|------|------|
| SyncByte | 2 Byte | 固定包头 `0x4E66` |
| Length | 2 Byte | 数据长度（不含包头、命令字、CRC），16进制 0x0000~0xFFFF |
| Command | 2 Byte | 命令字 |
| data | Length | 数据段（JSON 格式） |
| CRC | 4 Byte | CRC32 校验（不含包头） |

## 查询状态

**命令字：** `0x1E00`

查询机器人实时状态数据，支持单次应答和持续推送两种模式。

### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| robot | int | 是 | 机器人编号，范围 [1, 4] |
| channel | int | 否 | 通道号，最多 9 个通道，默认为 1 |
| mode | int | 否 | 回复模式：`0`=应答（请求一次回复一次），`1`=持续（持续回复） |
| interval | int | 否 | 持续回复间隔（ms），仅 `mode=1` 时有效，范围 [1, 60000] |
| stop | int | 否 | 停止发送：`1`=停止当前通道的持续发送 |
| queryType | string[] | 是 | 查询参数类型列表 |
| typeCfg | object | 否 | 查询内容详细配置 |

#### queryType 可选值

| 值 | 说明 |
|----|------|
| realPosACS | 实时坐标位置（关节坐标系） |
| realPosMCS | 实时坐标位置（直角坐标系） |
| realPosPCS | 实时坐标位置（工具坐标系） |
| realPosUCS | 实时坐标位置（用户坐标系） |
| axisVel | 轴速度 |
| axisAcc | 轴加速度 |
| torque | 力矩 |
| electric | 电流 |
| numGVar | 数值变量 |
| IO | IO 端口 |
| serialNum | 序列号 |
| runControlPosRunNum | 运动控制当前轨迹已运行点位 |
| detailedMotionPos | 详细运动点位（每个机器人同一时刻仅支持一个通道查询） |

#### typeCfg 配置

| 参数 | 类型 | 说明 |
|------|------|------|
| numGVar | string[] | 查询指定全局变量，如 `["GD001", "GA001", "GI001"]` |
| IO | string[] | 查询指定 IO，如 `["DI1", "DI16", "DO1"]` |
| detailedMotionPos | object | 详细运动参数配置 |
| detailedMotionPos.num | int | 每帧数据回复的点位数目 |
| detailedMotionPos.optional | string[] | 可选参数：`ACS`（关节坐标）、`MCS`（直角坐标）、`time`（时间戳）、`reset`（重置点位记录） |

> **注意：** 查询的 IO 若不存在，返回 `-1`；全局变量类型：`GA`=bool，`GI`=int，`GD`=double。

### 请求示例

```json
{
  "robot": 1,
  "channel": 1,
  "mode": 0,
  "queryType": ["realPosACS", "realPosMCS", "torque", "IO"],
  "typeCfg": {
    "numGVar": ["GD001", "GA001", "GI001", "GI333"],
    "IO": ["DI1", "DI16", "DO1", "DO3", "DO17"],
    "detailedMotionPos": {
      "num": 10,
      "optional": ["ACS", "MCS", "time", "reset"]
    }
  }
}
```

### 响应参数

| 参数 | 类型 | 说明 |
|------|------|------|
| robot | int | 机器人编号 |
| channel | int | 通道号 |
| replyData | object | 查询结果，未查询的字段不会返回 |

#### replyData 字段

| 字段 | 类型 | 说明 |
|------|------|------|
| realPosACS | double[] | 关节坐标系坐标：机器人（前7个）+ 外部轴（后若干） |
| realPosMCS | double[] | 直角坐标系坐标 |
| realPosPCS | double[] | 工具坐标系坐标 |
| realPosUCS | double[] | 用户坐标系坐标 |
| axisVel | double[] | 轴速度 |
| axisAcc | double[] | 轴加速度 |
| torque | double[] | 力矩 |
| electric | double[] | 电流 |
| numGVar | object | 全局变量，格式 `{"变量名": 值}` |
| IO | object | IO 状态，格式 `{"IO名": 状态值}` |
| runControlPosRunNum | int | 当前轨迹已运行点位数目 |
| serialNum | int | 序列号，每次回复加 1，每个通道独立 |
| detailedMotionPos | object | 详细运动点位 |
| detailedMotionPos.num | int | 本帧点位数目 |
| detailedMotionPos.data | array | 点位详细数据 |

#### detailedMotionPos.data 每项

| 字段 | 类型 | 说明 |
|------|------|------|
| time | int[] | 时间戳 `[秒, 纳秒]` |
| ACS | double[] | 关节坐标 |
| MCS | double[] | 直角坐标 |

### 响应示例

```json
{
  "robot": 1,
  "channel": 1,
  "replyData": {
    "realPosACS": [1.1, 1.2, 1.3, 1.4, 1.5, 1.6, 1.7, 2.1, 2.2, 2.3, 2.4, 2.5],
    "realPosMCS": [100.5, 200.3, 300.1, 45.0, -90.0, 180.0, 0.0],
    "torque": [10.5, 11.2, 9.8, 8.5, 7.2, 6.1, 5.0],
    "electric": [1.2, 1.3, 1.1, 0.9, 0.8, 0.7, 0.6],
    "numGVar": {"GD001": 0.1, "GA001": true, "GI001": 33, "GI333": -55},
    "IO": {"DI1": 1, "DI16": 0, "DO1": 0, "DO3": 1, "DO17": -1},
    "serialNum": 1
  }
}
```

## 运动控制

**命令字：** `0x1E02`

控制机器人运动，支持独立点位和连续轨迹两种模式。

> **前提条件：** 运动前需将伺服设置为使能状态，否则机器人不会动作。

### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| robot | int | 是 | 机器人编号，范围 [1, 4] |
| clearBuffer | int | 否 | 清空轨迹缓存：`1`=清空，`0`=不清空 |
| targetMode | int | 否 | 运动模式：`0`=独立点位，`1`=连续轨迹 |
| extMove | int | 否 | 外部轴运动：`0`=无，`1`=有 |
| sync | int | 否 | 外部轴同步：`0`=不同步，`1`=同步 |
| cfg | object | 否 | 运动参数配置 |
| targetPos | double[] | 否 | 单目标点（使用此字段只走一个点） |
| targetVec | object[] | 否 | 多目标点（按顺序走完所有点） |

#### cfg 运动参数

| 参数 | 类型 | 说明 |
|------|------|------|
| coord | string | 坐标系：`ACS`（关节）、`MCS`（直角） |
| speed | int | 速度，不填使用默认值 |
| acc | int | 加速度，不填使用默认值 |
| pl | int | 速度平滑参数，仅独立点位+targetVec时有效，默认5 |
| moveMode | string | 运动模式：`TeachSet`、`MOVJ`、`MOVL`、`MOVS`，默认 `TeachSet` |
| extMove | int | 外部轴运动：`0`=无，`1`=有 |
| sync | int | 外部轴同步：`0`=不同步，`1`=同步 |
| timeout | int | 超时时间（ms），仅连续轨迹模式有效，范围 [10, 10000] |

#### targetVec 每项

| 字段 | 类型 | 说明 |
|------|------|------|
| pos | double[] | 目标点坐标：机器人（前7个）+ 外部轴（后若干） |

> **注意：** `targetPos` 和 `targetVec` 同时存在时，仅 `targetPos` 生效。

### 请求示例（独立点位）

```json
{
  "robot": 1,
  "clearBuffer": 1,
  "targetMode": 0,
  "cfg": {
    "coord": "ACS",
    "extMove": 0,
    "sync": 0,
    "speed": 100,
    "acc": 100,
    "pl": 5,
    "moveMode": "TeachSet"
  },
  "targetPos": [1.1, 1.2, 1.3, 1.4, 1.5, 1.6, 1.7, 2.1, 2.2, 2.3, 2.4, 2.5]
}
```

### 请求示例（连续轨迹）

```json
{
  "robot": 1,
  "clearBuffer": 1,
  "targetMode": 1,
  "cfg": {
    "coord": "ACS",
    "extMove": 0,
    "sync": 0,
    "speed": 100,
    "acc": 100,
    "moveMode": "MOVL"
  },
  "targetVec": [
    {"pos": [1.1, 1.2, 1.3, 1.4, 1.5, 1.6, 1.7, 2.1, 2.2, 2.3, 2.4, 2.5]},
    {"pos": [2.1, 2.2, 2.3, 2.4, 2.5, 2.6, 2.7, 3.1, 3.2, 3.3, 3.4, 3.5]},
    {"pos": [3.1, 3.2, 3.3, 3.4, 3.5, 3.6, 3.7, 4.1, 4.2, 4.3, 4.4, 4.5]}
  ]
}
```

### 响应参数

| 参数 | 类型 | 说明 |
|------|------|------|
| robot | int | 机器人编号 |
| success | bool | 是否接收成功 |
| size | int | 轨迹缓存队列当前轨迹数目（仅 success=true 时返回） |
| cause | string | 失败原因，为空表示成功 |

#### cause 错误原因

| 值 | 说明 |
|----|------|
| (空) | 接收成功 |
| busy | 当前有未传输完成的数据 |
| timeout | 接收超时 |
| dataErr | 数据错误 |
| termination | 发送端终止了正在传输的数据 |

### 响应示例

```json
{
  "robot": 1,
  "success": true,
  "size": 1
}
```

## 变量、IO 控制

**命令字：** `0x1E04`

查询和设置全局变量以及 IO 端口状态。

### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| numGVar | object | 否 | 查询变量，格式 `{"变量名": 值}` |
| IO | object | 否 | 查询 IO，格式 `{"IO名": 状态值}` |

> 变量类型前缀：`GA`=bool，`GI`=int，`GD`=double

### 请求示例

```json
{
  "numGVar": {"GA001": true, "GI033": 6, "GD666": -2.33},
  "IO": {"DO1": 1, "DO13": 0}
}
```

### 响应参数

返回当前变量值和 IO 状态。

```json
{
  "numGVar": {"GA001": true, "GI033": 6, "GD666": -2.33},
  "IO": {"DO1": 1, "DO13": 0}
}
```

## 开关伺服点位运动控制

**命令字：** `0x1E10`

开启或关闭伺服点位运动控制模式。

> **注意：** 此功能与「运动控制」功能不可同时使用。

### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| robot | int | 是 | 机器人编号，范围 [1, 4] |
| switch | bool | 是 | 开关：`true`=开启，`false`=关闭 |

### 请求示例

```json
{
  "robot": 1,
  "switch": true
}
```

### 响应参数

| 参数 | 类型 | 说明 |
|------|------|------|
| success | bool | 操作是否成功 |
| cause | string | 失败原因，为空表示成功 |

#### cause 错误原因

| 值 | 说明 |
|----|------|
| (空) | 成功 |
| busy | 当前通道被占用 |
| dataErr | 数据错误 |
| startupErr | 启动失败 |

### 响应示例

```json
{
  "success": true
}
```

## 点位发送

**命令字：** `0x1E12`

在伺服点位运动控制模式下，发送点位数据到控制器。

### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| robot | int | 是 | 机器人编号，范围 [1, 4] |
| end | int | 否 | 停止传输：`1`=停止持续传输 |
| sum | int | 否 | 总帧数（连续传输模式） |
| count | int | 否 | 当前帧序号（连续传输模式） |
| PosVec | double[][] | 否 | 点位坐标数组，每项为 [J1, J2, J3, J4, J5, J6, J7, E1, E2, E3, E4, E5] |

> **说明：** 坐标格式为机器人（前7个关节）+ 外部轴（后5个）。点位未传输完毕时再次发送会清空之前的数据。

### 请求示例

```json
{
  "robot": 1,
  "end": 1,
  "sum": 10,
  "count": 1,
  "PosVec": [
    [1.1, 1.2, 1.3, 1.4, 1.5, 1.6, 1.7, 2.1, 2.2, 2.3, 2.4, 2.5],
    [2.1, 2.2, 2.3, 2.4, 2.5, 2.6, 2.7, 3.1, 3.2, 3.3, 3.4, 3.5]
  ]
}
```

### 响应参数

| 参数 | 类型 | 说明 |
|------|------|------|
| robot | int | 机器人编号 |
| success | bool | 是否接收成功 |
| cause | string | 失败原因，为空表示成功 |

#### cause 错误原因

| 值 | 说明 |
|----|------|
| (空) | 接收成功 |
| notStart | 未开启伺服点位运动控制模式 |
| dataErr | 数据错误 |
| termination | 发送端终止了正在传输的数据 |
| cacheFull | 缓存区已满（最大缓存 6 条轨迹） |

### 响应示例

```json
{
  "robot": 1,
  "success": true
}
```
