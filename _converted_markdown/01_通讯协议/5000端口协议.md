# 5000端口协议

5000 端口主要用于控制器与上位机之间的**文件上传、下载和备份**操作。

---

## 上位机请求上传文件

**命令字：** `0x1D00`

请求将文件上传到控制器。

### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| name | string | 是 | 上传文件的文件名 |
| size | int | 是 | 文件大小（字节） |

### 请求示例

```json
{
  "name": "vxWorks",
  "size": 4096
}
```

### 控制器回复

**命令字：** `0x1D01`

| 参数 | 类型 | 说明 |
|------|------|------|
| answer | string | `yes`=同意上传，`busy`=控制器忙碌 |

### 响应示例

```json
{
  "answer": "yes"
}
```

---

## 上位机向控制器上传文件

**命令字：** `0x1D04`

文件内容传输（纯二进制数据，非 JSON）。

---

## 上位机告诉控制器上传结束

**命令字：** `0x1D05`

通知控制器文件上传已完成。

---

## 控制器回复确认接收成功

**命令字：** `0x1D06`

### 响应参数

| 参数 | 类型 | 说明 |
|------|------|------|
| finish | bool | `true`=上传成功，`false`=上传失败 |

### 响应示例

```json
{
  "finish": true
}
```

---

## 上位机请求下载文件

**命令字：** `0x1D02`

请求从控制器下载文件。

### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| name | string | 是 | 作业文件在控制器中的完整路径 + 文件名 |

> **示例：** `job/R1/WWWWW.JBR`

### 请求示例

```json
{
  "name": "job/R1/WWWWW.JBR"
}
```

### 控制器回复下载请求

**命令字：** `0x1D03`

| 参数 | 类型 | 说明 |
|------|------|------|
| answer | string | `yes`=同意下载，`busy`=控制器忙碌 |
| name | string | 文件名（回显） |
| size | int | 文件大小（字节） |

### 响应示例

```json
{
  "answer": "yes",
  "name": "job/R1/WWWWW.JBR",
  "size": 217
}
```

---

## 控制器向上位机发送下载文件

**命令字：** `0x1D07`

控制器发送文件内容（纯二进制数据）。

---

## 上位机回复接收结果

**命令字：** `0x1D08`

确认文件接收状态。

---

## 控制器告诉上位机下载完成

**命令字：** `0x1D09`

---

## 上位机回复确认接收成功

**命令字：** `0x1D0A`

### 响应参数

| 参数 | 类型 | 说明 |
|------|------|------|
| finish | bool | `true`=下载成功，`false`=下载失败 |

### 响应示例

```json
{
  "finish": true
}
```

---

## 下载连接断开

**命令字：** `0x1D0B`

断开文件传输连接。

---

## 备份操作

### 上位机请求备份文件

**命令字：** `0x1D10`

请求备份控制器数据。

### 控制器回复备份请求

**命令字：** `0x1D11`

| 参数 | 类型 | 说明 |
|------|------|------|
| answer | string | 备份文件名 |

### 响应示例

```json
{
  "answer": "controller.tar.gz"
}
```

### 上位机发送下载 log 文件完成

**命令字：** `0x1D12`

### 上位机发送下载 eni 文件完成

**命令字：** `0x1D13`
